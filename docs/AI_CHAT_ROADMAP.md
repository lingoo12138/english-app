# AI 对话陪练 — 进阶功能需求池

> 状态:**待设计**(用户没想好具体怎么做)
> 最后更新: 2026-07-21

## 现有功能(v0.11)

- 5 场景(咖啡店/机场/购物/酒店/会议) + 6 难度(A1-C2)
- 走 LLM 多渠道(OpenAI 协议统一)
- TTS 朗读回复
- Mock 渠道零成本测试
- 对话历史仅内存,刷新丢失

## 进阶候选(按价值排序)

### 1. 🎤 语音输入(高价值,优先)

- **目标**: 取代键盘输入,像真人对话
- **实现**: MediaRecorder 录 5-10s 音频 → 转 base64 → LLM Vision/Whisper → 转文字
- **依赖**: LLM 渠道(OpenAI Whisper / Gemini 音频)或独立 STT 渠道
- **UI**: 输入区加 🎙 按钮,按住录音,松手发送
- **场景**: 练发音 + 测理解双管齐下

### 2. 💾 对话持久化 + 历史记录

- **目标**: 关闭页面不丢,能复习
- **实现**: IndexedDB 存 `chats` 表(id, scenario, level, messages[], ts)
- **UI**: 加 `/chat/history` 页面(列表 + 进入继续)或聊天页左侧历史抽屉
- **数据**: 每条对话 ~2KB,1 年用 1000 次 = 2MB(可接受)
- **导出**: 支持导出为 PDF / Markdown 复习

### 3. 📊 学习报告 + 错题分析

- **目标**: 显示学了什么、哪里卡
- **数据点**:
  - 各场景使用次数 + 平均消息轮次
  - 高频用户表达错误类型(语法 / 词汇 / 时态)
  - LLM 纠正次数 + 错误样本
- **UI**: `/chat/report` 页面 + 周报
- **价值**: 让用户看到进步,提高粘性

### 4. 🔄 多角色 / 多人对话

- **目标**: 模拟多人场景(如点菜时朋友+服务员+你)
- **实现**: System prompt 指定多个 NPC,轮流回复
- **场景**: 团队会议、朋友聚会、家庭聚餐
- **UI**: 多角色头像 + 不同颜色气泡

### 5. 🎯 自由话题 / 角色扮演

- **目标**: 不限场景,自定义角色
- **实现**: 用户输入"我想和乔布斯聊创业" → 动态 prompt
- **UI**: 高级模式开关 + textarea 输入角色/话题

### 6. 🌐 实时纠错 + 提示

- **目标**: 用户说错时立即给提示,而不是累积到最后
- **实现**:
  - 方式 A: 用户每发一条,后台另起一个 LLM call 做"纠错审查"
  - 方式 B: System prompt 要求 LLM 每条回复末尾加 [纠正: ...]
- **UI**: 用户气泡下面加红色小标签 + "为什么错" 展开

### 7. 🗣 发音评测

- **目标**: 语音输入时给发音打分
- **实现**: 复用现有 PronunciationPractice 组件
- **集成**: 跟 #1 语音输入一起做

### 8. 📚 对话中单词收藏

- **目标**: 看到生词直接收藏,带上下文例句
- **实现**: 长按消息中单词 → 弹收藏窗 → 写 favorites 表
- **UI**: 单词高亮 + 翻译 tooltip

### 9. 🌙 离线对话(PWA 缓存)

- **目标**: 飞机模式也能练
- **实现**: 预生成常见对话脚本(每个场景 50 套),本地随机抽取
- **本质**: 扩展 Mock 渠道,加脚本库

### 10. 🧪 难度自适应

- **目标**: 不让用户手动选,LLM 自动调难度
- **实现**: 监测用户表达平均长度 / 词汇复杂度,动态调整 system prompt
- **算法**: 简单规则(消息长度 < 5 词 → 降级;> 15 词 → 升级)

## 待决问题(用户思考)

1. **数据归属**: 对话历史算学习记录吗?清空数据是否包括?
2. **隐私**: 对话内容是否本地?还是默认匿名上传 LLM?
3. **付费策略**: 高级报告 / 永久历史是否会员制?
4. **离线策略**: 是否支持"无 LLM 也能练"?
5. **多模态顺序**: 先做语音输入还是先做发音评测?

## 实现顺序建议

按 ROI 排序:

1. **#1 语音输入** + #7 发音评测(共用录音链路,一次实现双功能)
2. **#2 对话持久化**(让功能"留下来",不流失)
3. **#6 实时纠错**(核心学习价值)
4. **#3 学习报告**(留存 + 商业化潜力)
5. 其余按需

## 技术依赖

- 新 STT 渠道(Whisper / 阿里云语音 / 讯飞)
- 录音组件复用现有 PronunciationPractice
- IndexedDB 新表 chats
- LLM streaming(流式输出,体验更好)
- Web Worker 跑 LLM 防阻塞 UI(本地 LLM 场景)
