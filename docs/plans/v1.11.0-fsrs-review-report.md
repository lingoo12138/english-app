## 3 个并行 Producer

### P1: v1.11.0-A FSRS 间隔重复算法升级 (B8, 1d → 0.5d)
- **Why**: SM-2 老 (1987年), FSRS 是 2023 开源新算法, 4 参数更精确
- **Scope**:
  - `src/lib/fsrs.ts` (~120 行): FSRS 4.5 算法简化版
    - 4 个参数: D (难度) / S (稳定性) / R (可检索性) / T (间隔)
    - 4 个评级: Again (1) / Hard (2) / Good (3) / Easy (4) — 复用 Anki 4 档
    - `initCard()`: 新卡片初始化
    - `reviewCard(card, rating, now)`: 评级后更新参数 + 计算下次 due
    - `getRetrievability(card, now)`: 当前可检索概率
  - **不替换现有 SM-2** (避免破坏 v0.22.9 Anki), **新加并平行支持**
    - plan.ts 加 `useFSRS` 标志 (默认 false, 保持 SM-2)
    - 旧数据用 SM-2, 新数据用 FSRS (向后兼容)
  - `tests/fsrs.test.ts` (6 测试):
    - initCard 默认值
    - reviewCard Again → 短期重试
    - reviewCard Good → 拉长间隔
    - reviewCard Easy → 更大间隔
    - getRetrievability 随时间下降
    - SM-2 vs FSRS 转换 (迁移)

### P2: v1.11.0-B 复习中心智能队列 (B10, 1d → 0.5d)
- **Why**: ReviewCenter 现在按时间排, 改成"价值排" (due 早 + 难词 + 新词)
- **Scope**:
  - `src/lib/reviewQueue.ts` (~80 行):
    - `scoreReviewItem(item, now)`: 返回优先级分数
    - due 早: 分数越高 (越早越前)
    - 难词 (ease 低): 加分
    - 新词 (前 3 次): 加分 (前期多见)
    - `sortReviewQueue(items, now)`: 按分数排序
  - `src/pages/ReviewCenter.tsx` 改造:
    - 现有 `dueItems` 改用 `sortReviewQueue` 排序
    - 顶部加 "🎯 智能排序" 切换 (默认开)
    - 不破坏现有 review 流程
  - `tests/reviewQueue.test.ts` (5 测试):
    - scoreReviewItem due 早
    - scoreReviewItem 难词加分
    - scoreReviewItem 新词加分
    - sortReviewQueue 降序
    - 智能排序 vs 时间排序 输出不同

### P3: v1.11.0-C 学习日报/周报 (B11, 1d → 0.5d)
- **Why**: "晒朋友圈" + 留存情感化设计
