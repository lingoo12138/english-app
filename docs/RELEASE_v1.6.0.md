## 🐛 修复的 Bug

### P1 必修 (7)

| # | 模块 | Bug | 影响 |
|---|------|-----|------|
| 1 | 写作批改 | 切回 "批改" tab 时输入文本被清空 | 用户输入 600 字 → 切到 history 再切回 → **全丢** |
| 2 | 写作批改 | 点历史记录"打开"按钮加载内容被 useEffect 覆盖 | **加载无效**, 仍是空白 |
| 3 | 听力 | DictationMode 切换课程时听写答案残留 | 答了一半的课切换, 旧答案还在 |
| 4 | 听力 | QuestionsMode 切换课程时答案残留 | 同上 |
| 5 | 错题讲解 | 按钮永远不显示 "加载中..." | 用户不知道在等, 体验差 |
| 6 | 短语用法 | 同上 | 同上 |
| 7 | AI 对话 | 语音输入累积无上限 | 说话久了 input 超过 LLM token 上限 → 报错 |

### P2 选修 (6)

- 写作批改: 截断 500 字符时旧 input 仍发给 LLM (用 text 变量修复)
- 听力: 听写/答题播放按钮重复点击 (加 if playing return)
- 短语用法: 缓存解析失败时 tip 显示 JSON 字符串 (改 "暂无数据")
- 写作批改 + AI 对话: 3 处 `catch (e: any)` 改 `unknown + Error 守卫` (v1.5 review 漏修)
- 写作批改 parseResult: `(e: any)` 参数 + 类型断言改 `unknown + WritingErrorType` 类型
